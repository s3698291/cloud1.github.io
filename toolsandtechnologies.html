As for the cybersecurity part of our project idea, a simple and common yet secure security method will be used. Many systems has used the two-factor authenticator to protect users from getting hacked. It requires the user enter another code that will be sent to their emails or phone when logging in. The code generated will be a complicated alphanumeric eight digit code that is random. This provides an extra protection against password theft. Other than that, the data will be encrypted when it is travelling between the device and the server. It is protected with a TLS/SSl tunnel using 128-bits AES. It’s not the best AES encryption algorithm (there’s the AES 256-bit), but it’s safe for the foreseeable future. In simple terms, it means that when the user decides to store information onto Cloud 1, the device will connect to a proxy server and send the data to the server through the proxy server. This way there will be a lower risk of data leak. Normally, the service will keep a copy of the user's’ password but it will mean that there is a possibility that other people will be able to access the data. To further the security, Cloud 1 employs the zero-knowledge encryption method. This means that no one excluding the user will have access for the the keys to their data, not even Cloud 1. Zero-knowledge encryption or private encryption may be a good encryption method but there are some downsides to it. Because only the user has the keys, if they forget the key, no one will be able to access the data. The data will be gone forever, trapped behind a digital wall with no method of access to get it back. Besides that, it is also slower than normal encryption. Cloud 1 will also implement a security system where the decryption key changes every 3 days. Furthermore, there is an activity log that shows what was stored, what was taken out, when and where. Both Cloud 1 and the user will have a copy of the activity log.
